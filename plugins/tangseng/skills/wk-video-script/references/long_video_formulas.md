# 中长视频脚本公式参考

> 中长视频有空间展开，但核心还是**节奏紧凑、不说废话**。比短视频更多细节和深度，但同样拒绝拖沓。

## 中长视频结构公式：起承转合

```
起（抛出痛点，引出主角）→ 承（介绍背景，铺垫基础）→ 转（核心干货，层层拆解）→ 合（总结互动，闭环收尾）
```

**起承转合时间分配**（约4分钟视频）：
- **起**：0:00-0:30（30秒）——抛出痛点，引出主角
- **承**：0:30-1:00（30秒）——介绍背景，铺垫基础
- **转**：1:00-3:30（2分30秒）——核心干货，层层拆解
- **合**：3:30-4:30（1分钟）——总结互动，闭环收尾

---

## 中长视频开头：起承转合的"起"

**开头任务**：用30秒建立"这个问题跟我有关"的连接，然后抛出一个"然后呢？"的钩子。

好的"起"的结构：
1. 抛出痛点/共鸣点（10秒）
2. 建立"我懂你"的关系（10秒）
3. 给出视频的价值承诺（10秒）
4. 抛出入主题/主角

---

## 中长视频节奏要求：起承转合的"承"与"转"

**"承"的节奏**：
- 平稳铺垫，不要急于给答案
- 建立期待感，让观众想知道"然后呢"
- 30秒内必须进入"转"段

**"转"的节奏**：
- 每2-3分钟设置一个小高潮，不能平铺直叙
- 每个小节要有结论，不要堆信息
- 细节要具体：具体的数字、场景、案例
- 保持口语感，主动打断自己制造悬念

**"合"的节奏**：
- 给出明确的行动呼吁或金句
- 引导互动（评论/关注/转发）
- 闭环收尾，呼应开头

---

## 中长视频禁止

- 禁止教科书式开场（"今天给大家分享..."）
- 禁止没有观点的叙述（光讲事实不讲判断）
- 禁止冗长的背景铺垫（"承"段不超过30秒）
- 禁止突然严肃或突然松垮（保持风格一致）

---

## 中长视频类型专项公式

你的视频类型主要是两类：**干货知识分享类** 和 **软件技术测评类**。以下是针对这两类的专项起承转合公式：

---

### 类型一：干货知识分享类

**核心特点**：传授知识、分享经验、启发思考

**起承转合公式**：

| 段落 | 时长 | 任务 | 关键动作 |
|------|------|------|---------|
| **起** | 0:00-0:30 | 抛出痛点，引出主题 | 共情 + 场景化描述 |
| **承** | 0:30-1:00 | 介绍背景，铺垫知识 | 建立"我懂你"的连接 |
| **转** | 1:00-3:30 | 核心干货，层层拆解 | 每个知识点配一个案例 |
| **合** | 3:30-4:30 | 总结回顾，行动呼吁 | 给出具体动作 + 互动引导 |

**"转"段展开方式**（2分30秒）：
- 知识点1（40秒）：核心概念 + 生活场景类比
- 知识点2（40秒）：实用技巧 + 亲身经历
- 知识点3（40秒）：常见误区 +避坑指南
- 升华总结（30秒）：连接更大的认知/价值观

**案例参考**：
- "追问式工作流"视频：痛点→背景→3个追问技巧→行动呼吁
- "AI时代生存指南"视频：焦虑现状→认知重建→3个方法→互动结尾

---

### 类型二：软件技术测评类

**核心特点**：评测工具、展示功能、客观分析

**起承转合公式**：

| 段落 | 时长 | 任务 | 关键动作 |
|------|------|------|---------|
| **起** | 0:00-0:30 | 制造悬念，引出工具 | 痛点/对比/反差 |
| **承** | 0:30-1:00 | 介绍背景，建立期待 | 突出差异化优势 |
| **转** | 1:00-3:30 | 功能演示，详细测评 | 逐个功能展示 + 效果对比 |
| **合** | 3:30-4:30 | 总结评价，互动引导 | 优缺点总结 + 适用人群 + 互动 |

**"转"段展开方式**（2分30秒）：
- 核心功能1（50秒）：演示 + 效果 + 适用场景
- 核心功能2（50秒）：演示 + 效果 + 适用场景
- 核心功能3（40秒）：演示 + 效果 + 适用场景
- 优缺点总结（10秒）：3个优点 + 2个缺点

**"合"段公式**：
```
总结一句话：「总体来说，这个工具适合XXX，不适合XXX」
互动引导：「评论区告诉我，你想看什么工具的测评？」
```

**案例参考**：
- "OpenClaw智能体测评"视频：痛点→介绍→5大功能→上手3步→总结互动
- "Claude Code体验"视频：悬念→背景→核心场景演示→优缺点→引导关注
